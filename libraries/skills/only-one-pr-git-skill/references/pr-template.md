# Pull Request Template

The GitHub PR body MUST be English only and MUST use this exact Markdown structure. Write `N/A` for items that do not apply.

## 🎯 Objective
- (Briefly summarize the new feature, bug fix, or purpose)
- (Explain WHY this change is necessary)

## 🔗 Related Issues
- Closes # (Extract Issue/Ticket ID from current branch if possible, otherwise `N/A`)

## 🛠️ Technical Changes
- (List the main changes in logic, components, or file structure)
- (Briefly explain complex code or important design decisions)

## ⚠️ Impact & Environment
- **New Dependencies:** (List packages added to package manifests, or `None`)
- **Environment Variables:** (List `.env` changes, or `None`)
- **Database Changes:** (List schema/migration changes, or `None`)

## 📋 Test Cases Coverage
- **Happy Paths:** (List scenarios where everything works perfectly)
- **Edge/Negative Cases:** (List validation, error handling, or boundary cases)

## 🧪 How to test
- (Provide step-by-step local testing instructions)

## 📸 Screenshots
- *Please drag and drop screenshots or video recordings here if there are UI changes.*

# Vietnamese Summary Rule

The chat summary may be Vietnamese, but it must never be included in the GitHub PR body.
